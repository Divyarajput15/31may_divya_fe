what are the html entities ? with example

HTML entities are special codes used to represent reserved characters in HTML. These entities are necessary because certain characters, such as the less than sign (<) or greater than sign (>), can be confused with HTML tags if they are not properly encoded. By using HTML entities, these characters can be displayed as intended without affecting the structure of the HTML code. There are two ways to represent HTML entities: using an entity name or using an entity number. An entity name is a descriptive name enclosed in ampersand (&) and semicolon (;), while an entity number is a numeric value preceded by an ampersand (&) and hash (#) symbol. For example, to display a less than sign (<), we can use the entity name `<` or the entity number `<`. similarly, the greater than sign (>) can be represented as `>` or `>`. Here are some commonly used HTML entities: - Non-breaking space: ` ` or ` ` - represents a space that will not break into a new line. - Ampersand: `&` or `&` - represents the ampersand character itself. - Double quotation mark: `"` or `"` - represents a double quotation mark. - Single quotation mark (apostrophe): `'` or `'` - represents a single quotation mark. - Cent sign: `¢` or `¢` - represents the cent currency symbol. - Pound sign: `£` or `£` - represents the pound currency symbol. - Yen sign: `¥` or `¥` - represents the yen currency symbol. - Euro sign: `€` or `€` - represents the euro currency symbol. - Copyright symbol: `©` or `©` - represents the copyright symbol. - Registered trademark symbol: `®` or `®` - represents the registered trademark symbol. By using HTML entities, you can ensure that special characters are correctly displayed in HTML documents without causing any parsing or rendering issues.
